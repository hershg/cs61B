Lecture 02 - 01/20

Using and Defining Classes

  • Compilaton
    • javac is compiler
    • java is interpreter
  • Defining and Instantiating classes
    • To run a class, we need a main method
    • Not all classes have main method (ex: some class called from another class; other class has the main method)
    • Classes also have data such as instance variables
    • Classes can be instantiated as objects
    • For passing in parameters to a class, need a constructor that takes in the parameters and appropriately deals with them
    • Creating arrays in Java:
      Dog[] dogs = new Dog[4]
    • Polymorphism (methods have same name but take in different parameters -> different functionality)

  • Closer look at static
    • Non-static methods, aka instance methods, can look at instance variables, while static methods cannot
    • Why Static methods? Some classes are never instantiated, so for example
      x = Math.round(5.6)
      is much nicer than
      Math m = new Math();
      x = m.round(x)
    • Variables/methods defined in a class are all members of the class
    • Static members are accessed using class name
    • Non-static members are accessed using instance name; can't be accessed through class name
    • Static methods must access instance variables via a specific instance (such as taking an instance as a parameter and accessing its instance variables)

  • public static void main(String[] args)
    • args refers to parameters called with main when the program is run (aka the command line arguments)
      • Ex: from command prompt:
        java myProgram arg0 arg1 arg2
  • Using libraries (e.g. StdDraw, In)
    • In 61B will use libraries such as
      • built-in Java libs (Math, String, Integer, List, Map, ...)
      • Princeton stdlib (StdDraw, StdAudio, In)
    • Important programming skill to know how to find and use libraries

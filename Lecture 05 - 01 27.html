Double Linked Lists (DLLists):

  • SLLists are big improvement on the simplistic IntList
    • access control (public/private), nested class, caching, generalizing (sentinel node represents empty list)
  • Problem: inserting link to SLList at back much slower than at front
  • Solution: give each link backwards and forward links. So each link has a prev pointing to previous link, next pointing to next element, and item which is the link's value. This is a doubly linked list - DLList; it is a circular linked list
    • The DLList has a sentinel variable pointing to the sentinel node, and a size variable caching the size of the linked list
    • The sentinel node now represents a null doubly linked list
    • A null DLList (one with only a sentinel node) has a prev pointer to itself and a next pointer to itself
  • We can make our linked lists generic and able to hold elements of other types because Java lets us defer type selection until declaration. We can define our SLList class with public class SLList<someType> {...} and then use someType throughout that class whenever dealing with the element type. We define/instantiate such a list with
    SLList<Integer> s1 = new SLList<>(5); or
    SLList<String> s2 = new SLList<>("hi");

Arrays

  • Array practice: http://goo.gl/nFYwzU
  • Goal over next few lectures is AList (Array list)
  • Array is Java object with a numbered sequence of memory boxes (whereas class instances have named memory boxes)
  • Index array A with A[i];
  • Arrays have fixed length n that can't change
  • All the n memory boxes (from 0 to n-1) have the same type of value (and same amount of bits)
  • int[] x = new int[3] creates an array with 3 int boxes
  • int[] y = new int[]{1,2,3,4,5}; declares, instantiates, and assigns an array
  • Can copy arrays with System.arraycopy(b,0,x,3,2); where b is source array, 0 is start position in source, x is target array, 3 is start position in target, 2 is amount to copy (python equivalent: x[3:5] = b[0:2])

Arrays vs Classes

  • Array 'boxes' accessed with [] notation, classe 'boxes' accessed with dot notation
  • Array 'boxes' all of same type
  • Class boxes can be of different types
  • Both have fixed number of boxes
  • array indicies can be computed at runtime; class member variales cannot be computed and used at runtime

Libraries, Abstract Classes, Interfaces, and Packages

  • So far in class, we have:
    • Created AList, SLList, and DLList list classes
    • Created a List61B list interface and made these list classes implement it
    • Then in projects we created ArrayDeque and LinkedListDeque, both of which implement the Deque interface
  • Other uses of interface inheritance:
    • Specify contract for common behavior shared by several data structures (such as Comparable for Dog objects in last lecture)
      • In Dog, implementing Comparable means it can be compared to other Dog objects
    • Provide a way to containerize common functions (such as the OffByN class that implements the CharacterComparator interface from project 1)
      • In projects, we had several different OffBy_ classes for comparing, all of which implement a common interface
      • As a result, we can containerize all of these under their common interface when we want to use a general type of CharacterComparator in various methods in our code
  • Abstract data types (ADT): types that are defined by their operations (interface) rather than implementation (for example, Deque is an ADT)
  • Very important interface in Java is Collection interface, which is for data types that are a collection of things
    • List, Set, and Map (maps are like Python dictionaries) are three major Java interfaces that all implement the Collection interface
    • All three of these are ADT's
    • Interfaces can implement other interfaces!
    • Java gives us several concrete classes that implement these interfaces/ADT's, such as:
      • ArrayList and LinkedList implement List
      • HashSet and TreeSet implement Set
      • HashMap and TreeMap implement Map
      • There are even more, such as stacks, queues, and priority queues

Java vs Python

  • In Python, sets, dictionaries, tuples, and lists are all "first class citizens" that have special syntax, and are very convenient for use to use
  • This is not the case with Java
  • While as a result this is makes Java much more verbose, it allows us to decide which implementation of a data type we use in Java (whereas Python already has chosen it for us)
    • Example: depending on application, may want alphabetically ordered TreeMap implementation, or faster but unordered HashMap implementation
  • Java has
    • Static types (type checking and guiding programmer)
    • Biased towards interface inheritance, making subtype polymorphism cleaner
    • Access control modifiers make abstraction barrier more solid
    • More efficient code due to control of engineering tradeoffs (which implementation type to use, ...), higher performance with single valued arrays

Interfaces and Abstract Classes

  • Every method of an interface must be public, so using this keyword on interface methods is redundant
  • So far, we have seen the following inheritances:
    • implements: Interface inheritance (what the class can do)
    • extends: Implementation inheritance (how the class can do it)
  • Interfaces:
    • Interfaces have abstract methods, meaning they cannot be instantiated
    • In interfaces, if we want to make a method concrete (give it implementation), we need to use the default keyword
    • Interfaces can provide variables, but these are public static final (can't change)
    • A class can implement/inherit from multiple interfaces
  • Abstract classes:
    • Abstract classes cannot be instantiated
    • Class is abstract if it is given the abstract keyword in signature
    • Methods of an abstract class without keyword are concrete
    • To make a method abstract (must be overridden by a subclass), use the abstract keyword
    • These can provide variables and methods of any kind (unlike interfaces which are all public and static final)
    • Subclasses must extend an abstract class
    • A class can extend/inherit from only one abstract class

Packages and Importing

  • We want to ideally give everything a canonical representation (unique, unchangeable)
  • We do this in Java by giving classes package names
  • A package is a namespace organizing classes/interfaces
  • We declare the package/namespace at the top of the code file
  • Naming convention is backwards
  • Ex: package com.godse.hersh;
  • We can use packages for importing with the import keyword
    • Ex: import com.godse.hersh.*;
    • Using the * character for imports imports all classes, but be careful with this (for example, if we do this and suddenly have multiple classes of same name and parameters, we will get error)
    • import static lets us import static members of a class
      • Ex: import static org.junit.Assert.*;

Below is a demo of the use of various Java collections

import java.util.List;

import java.util.ArrayList;

import java.util.Set;

import java.util.HashSet;

import java.util.Map;

import java.util.HashMap;



public class DemoCollections {



    public static String cleanString(String s) {

        return s.toLowerCase().replaceAll("[^A-Za-z0-9]", "");

    }



    public static List<String> getWords(String inputFilename) {

        In in = new In(inputFilename);



        List<String> w = new ArrayList<String>();

        while (!in.isEmpty()) {

            String rawString = in.readString();

            w.add(cleanString(rawString));

        }

        return w;

    }



    public static Set<String> findUniqueWords(List<String> words) {

        Set<String> ss = new HashSet<String>();

        for (String s : words) {

            ss.add(s);            

        }

        return ss;

    }



    public static Map<String, Integer> findWordCount(List<String> words, List<String> targets) {

        Map<String, Integer> wordCounts = new HashMap<String, Integer>();

        for (String s : targets) {

            wordCounts.put(s, 0);

        }



        for (String s : words) {

            if (wordCounts.containsKey(s)) {

                int oldCount = wordCounts.get(s);

                wordCounts.put(s, oldCount + 1);

            }            

        }



        return wordCounts;

    }



    public static void main(String[] args) {

        List<String> w = getWords("libraryOfBabylon.txt");

        List<String> targets = new ArrayList<String>();

        targets.add("lottery");

        targets.add("the");

        targets.add("babylon");

        System.out.println(findWordCount(w, targets));

    }

}

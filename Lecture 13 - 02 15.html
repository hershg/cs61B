Generics, Conversion, Promotion, Covariance

Generics:

Below is a demonstration of using the ArrayList class that takes in a generic data type (in this case, String):

ArrayList<String> L = new ArrayList<String>();
  • The second "String" on the right side at instantiation is no longer necessary since Java 8 (can just use a diamond operator instead now: <>)
  • Can't use primitive types as type arguments
    • Must use "Integer" instead of "int"
  • "Integer" is wrapper for "int" primitive type
  • At certain places, Java will use autoboxing/auto-unboxing: implicit conversions between wrappers and primitives. As a result, wrappers/primitives are almost interchangeable
    • Arrays are never autoboxed/unboxed (Integer[] and int[] are NOT interchangeable)
    • Autoboxing/unboxing has performance impact - wrappers take much more memory than primitives
    • Wrappers are mostly like a normal class
  • Also have primitive widening/promotion: moving from primitive type of narrow range to wider range
    • For example, "int" automatically converted to "double" (widened)
    • For other way, must cast "double" to "int"

Immutability:

  • An immutable data type is one for which an instance cannot change in any observable way after instantiation
    • ArrayDeque is mutable, Integer/String/Date all immutable
  • final keyword helps compiler make sure things are immutable
    • Declaring something final doesn't necessarily make it immutble (ex: if a final variable points to a mutable object)

ArrayMap

First, we create a Map61B interface in a Map61B package/folder:

package Map61B;

import java.util.List;



public interface Map61B<K, V> {

    /* Returns true if this map contains a mapping for the specified key. */

    boolean containsKey(K key);



    /* Returns the value to which the specified key is mapped. No defined

     * behavior if the key doesn't exist (ok to crash). */

    V get(K key);



    /* Returns the number of key-value mappings in this map. */

    int size();



    /* Associates the specified value with the specified key in this map. */

    void put(K key, V value);



    /* Returns a list of the keys in this map. */

    List<K> keys();

}

Now we make the ArrayMap class, that implements Map61B. The class also has some JUnit tests:

package Map61B;



import org.junit.Assert.*;

import java.util.List;

import java.util.ArrayList;

import org.junit.Test;



import static org.junit.Assert.*;



/**

 * An array based implementation of the Map61B class.

 */

public class ArrayMap<K, V> implements Map61B<K, V> {

    private K[] keys;

    private V[] values;

    int size;



    public ArrayMap() {

        keys = (K[]) new Object[100];

        values = (V[]) new Object[100];

        size = 0;

    }



    /** Returns the index of the given key if it exists,

     *  -1 otherwise. */

    private int keyIndex(K key) {

        for (int i = 0; i < size; i += 1) {

            if (keys[i].equals(key)) {

                return i;

            }

        }

        return -1;

    }



    public boolean containsKey(K key) {

        int index = keyIndex(key);

        return index > -1;

    }



    public void put(K key, V value) {

        int index = keyIndex(key);

        if (index == -1) {

            keys[size] = key;

            values[size] = value;

            size += 1;

            return;

        }

        values[index] = value;

    }



    public V get(K key) {

        int index = keyIndex(key);

        return values[index];

    }



    public int size() {

        return size;

    }



    public List<K> keys() {

        List<K> keylist = new ArrayList<K>();

        for (int i = 0; i < keys.length; i += 1) {

            keylist.add(keys[i]);

        }

        return keylist;

    }



    @Test

    public void test() {

        ArrayMap<Integer, Integer> am = new ArrayMap<Integer, Integer>();

        am.put(2, 5);

        int expected = 5;

        assertEquals((Integer) expected, am.get(2));

    }



    public static void main(String[] args) {

        ArrayMap<String, Integer> m = new ArrayMap<String, Integer>();

        m.put("horse", 3);

        m.put("fish", 9);

        m.put("house", 10);

    }

}

Generic Methods

Lastly, we create a class MaxHelper:

  • Note below how the MaxKey method only works if the keys are comparable. This is accomplished by making the key generic type K extend Comparable<K>. This process is called type upper bounds
  • To create generic methods, we define all of the generic type parameters in a diamond operator before the return type of the method
    • Ex: public static <K, V> V get(Map61B<K, V> sim, K key)
  • Finally, note the use of the wildcard character in the allBark method below. This ensures that the method only works if the key type passed in is of type Dog
    • The following method signatures are equivalent. The first is a generic method, and the second uses the wildcard operator:
      public static <V> void allBark(ArrayMap<Dog, V> am) {...}
      public static void allBark(ArrayMap<Dog, ?> am) {...}
    • We only use the wildcard operator if we don't care about the type for this parameter/we never reference it in the method
package Map61B;



import static org.junit.Assert.*;

import org.junit.Test;

import java.util.List;



/**

 * Class to demonstrate how generic methods work in Java.

 */

public class MapHelper {

    /* Write the following three methods:

    /* get(Key) : Return item in map if it exists. */



    public static <K, V> V get(Map61B<K, V> sim, K key) {

        if (sim.containsKey(key)) {

            return sim.get(key);

        }

        return null;

    }

    /* maxKey() : Returns max of all keys. Works only if x and y have comparable data. */

    public static <K extends Comparable<K>, V> K maxKey(Map61B<K, V> map) {

        List<K> keylist = map.keys();

        K largest = keylist.get(0);

        for (K k : keylist) {

            if (k.compareTo(largest) > 0) {

                largest = k;

            }

        }

        return largest;

    }



    /* allBark(): Makes all keys bark. Works only for keys of type Dog. */

    public static void allBark(ArrayMap<Dog, ?> am) {

        for (Dog d : am.keys()) {

            d.bark();     

        }

    }



    @Test

    public void testGet() {

        Map61B<String, Integer> m = new ArrayMap<String, Integer>();

        m.put("horse", 3);

        m.put("fish", 9);

        m.put("house", 10);

        Integer actual = MapHelper.get(m, "fish");

        Integer expected = 9;

        assertEquals(expected, actual);



        assertEquals(null, MapHelper.get(m, "awefawefawef"));

    }



    @Test

    public void testMaxKey() {

        Map61B<String, Integer> m = new ArrayMap<String, Integer>();

        m.put("horse", 3);

        m.put("fish", 9);

        m.put("house", 10);



        String actual = MapHelper.maxKey(m);

        String expected = "house";

        assertEquals(expected, actual);

    }    

}

Covariance:

  • Let's say we have a type FrenchDog that extends Dog
  • In the above definition of allBark, we would be able to use this function on ArrayMap<Dog, ?>, but not on ArrayMap<FrenchDog, ?>
  • This is because generic types are invariant
  • Arrays are covariant:
    • A FrenchDog "is-a" Dog
    • A FrenchDog[] "is-a" Dog[]
  • Because generic types are invariant, a List<FrenchDog> is NOT a List<Dog>
  • Two ways to fix this:
    1. Add a generic type to the method:
      public static <K extends Dog> void allBark(ArrayMap<K, ?> am) {

         for (Dog d : am.keys()) {

             d.bark();

         }

      }
    2. Add a bounded-wildcard:
      public static void allBark(ArrayMap<? extends Dog, ?> am) {

         for (Dog d : am.keys()) {

             d.bark();

         }

      }

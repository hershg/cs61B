Iteration and Exceptions

Exceptions

We throw exceptions explicitly with throw keyword, and provide useful information for user/for debugging. Example:

if (</*boolean check to throw the exception*/>) {

    throw new RuntimeException("Error Message");

}

We can also try some code and catch any exceptions:

  • This allows us to separate code for error handling from the actual main code
    try {

        //Try doing something

        ...

    } catch (Exception e1) {

        System.out.println("Got exception 1: " + e1);

        //Do something to fix exception 1

        ...

    } catch (Exception e2) {

        System.out.println("Got exception 2: " + e2);

        //Do something to fix exception 2

        ...

    }
  • Sometimes the compiler will force us to "catch"/"specify" certain exceptions. These are called "checked" exceptions
    • Subclasses of Error/RuntimeException are unchecked
    • All other Throwables are checked
    • For example, we can't just plainly throw a checked exception. We must instead "catch" the exception by surrounding it with a try/catch structure (try to throw the checked exception, catch if this fails)
    • We can also "specify" that an exception could be thrown with the throws keyword

      Catching (use when you're currently dealing with the exceptions):

      public static void main(String[] args) {

          try {

              //Code that can throw a checked exception 

              //Ex: we call a method that itself could call an exception

          } catch(IOException e) { //This is a type of checked exception

              //Do something

          }

      }

      Specifying (use when someone else should deal with the exceptions):

      public void someFunc() throws IOException, IndexOutOfBoundsException {

              //Code that can throw the specified checked exceptions

          }

      }

Iteration

  • This is the under-the-hood magic behind enhanced for loops.
  • For making a class/object iterable, we must:
    • Import Iterator
    • Make the class iterable by extending Iterable interface
      • This means it must have a public method iterator() that returns an Iterator instance
      • Some common Java structures may already extend this interface
    • Define the iterator method by returning an instance of an Iterator. This instance should override the hasNext and next methods of Iterator
  • Example:
import java.util.Iterator;



public class MyClass<T> extends Iterable<T> {

    ...



    public class MyIterator implements Iterator<T> {

        ...



        @Override

        public boolean hasNext() {

            ...

        }



        @Override

        public T next() {

            ...

        }

    }



    @Override

    public Iterator<T> iterator() {

        return new MyIterator();

    }



    ...

}
